Sobre esta API

A API Info Vacinas disponibiliza de forma padronizada serviços e funcionalidades oferecidos por este site referentes a consultas de locais de vacinação através de base uniforme que permitem ao usuário desta API realizar integração de suas aplicações ou automações com os serviços oferecidos pela API Info Vacinas.

Esta implantação faz parte do projeto info Vacinas que tem o objetivo de disponibilizar ao público residente no Brasil uma experiência de consulta a locais de vacinação com maior acurácia do que os existentes hoje em órgãos governamentais e seus canais oficiais.

Esta API foi construída a partir de conteúdo gratuito e tem fins didáticos e informacionais, não tendo fins lucrativos.

A base de dados disponibilizada por esta API foi construída a partir de cruzamento de bases oficiais do Governo Federal do Brasil (Gov.br) e transformada a partir do cruzamento de bases de geolocalização.

Atualmente a API InfoVacinas consta apenas com a versão inicial disponibilizada ao público, API infoVacinas 1.0

O versionamento de implementações respeitarão a Especificação de Versionamento Semântico (semVer)

Para versões futuras, as seguintes mudanças são esperadas e consideradas compatíveis:

  • Adição de novos recursos;
  • Adição de novos parâmetros opcionais a requisições a consultas existentes;
  • Adição de novos campos em respostas da API;
  • Alteração da ordem de campos;
  • Adição de novos elementos em enumerações.

Esta seção reúne erros que poderiam ser retornados por quaisquer endpoints listados na API infoVacinas.

A API infoVacinas retorna códigos de status HTTP a cada requisição indicando o sucesso ou as falhas. Dessa forma de maneira geral considere os seguintes códigos:

  • Códigos da série 200 a 2Xx equivalem sucesso na requisição;
  • Códigos da série 400 a 4Xx indicam falhas causadas pelas informações enviadas pelo cliente ou pelo estado atual das entidades;
  • Códigos da série 500 a 5xx indicam problemas no serviço no lado da API infoVacinas, podendo ser problemas temporários por indisponibilidades dos serviços de implementação utilizados.

Erros Gerais

Envio de Dados corretamente e resultados divergentes.

Nas transformações prepatratórias para formação desta base de dados, nas APIs consultadas de geolocalização retornou XXX resultados não encontrados no total de XXXXXX. Assim é possível que o pedido de consulta a API possam estar um destes resultados.

Parte dos dados das Unidades de Saúde obtidos junto a Gov.br não possuiam endereço completo, assim foram utilizadas estratégias para obter localização aproximada como uso de CEP e não a coordenada geográfica exata que seria possivel pelo logradouro mais númeração do local.
